    Unraid下使用Frpc的Docker安装实现自动启动

    使用过Unraid之后觉得很方便，但是在使用frp穿透的时候要么需要命令行执行，要么就是要用插件脚本实现，无奈本人文盲用起来太难，而我为了稳定是能不改Unraid主系统就不想去动，就研究下使用docker安装，这也是Unraid最大便捷之处，无奈网上没有教程，只有自己研究。
    docker商店也没有现成可以用的只有自己添加，学习了大神的讲解后基本了解，刚好frp客户端也比较简单，不需要目录也不需要端口，刚好适合我新手。

    前置

    服务器端和客户端配置就不说了，教程很多照做就好了
    由于Unraid应用商店没有我们就需要去docker网址搜索，因为我们只在unraid上安装客户端，所以搜索frpc就行了

    选择适合的，由于我服务器使用的最新的0.33的所以选择的更新最近，最受欢迎的snowdreamtech/frpc

    由于服务器安装的是0.33版本，不想unraid点击升级frpc的适合误点击，导致frpc升级而无法使用，所以需要找到确定的版本，点击tags，找到需要版本0.33.0右侧

    就是我们需要的docker参考地址+tags了

    正式操作

    docker页面，添加ADD CONTAINER

    点击最下方+ Add another Path, Port, Variable, Label or Device 映射ini文件位置

    目录配置是根据官方文档里的得出的，我们的目的是映射到appdata目录，好方便我们修改和保存
    docker run --restart=always --network host -d -v /etc/frp/frpc.ini:/etc/frp/frpc.ini --name frpc snowdreamtech/frpc

    在建立时候没有这个目录，可以通过WINSCP手动建立，复制frpc.ini后配置权限755
    也可以启动一下，目录也会自动建好，再将我们的frpc.ini复制到/mnt/user/appdata/frp/frpc/frpc.ini就行
    再重启应该就可以了

    有没有和服务器握手成功可以start后点击后方LOG查看，如果成功会和在命令行执行的时候一样，确定成功后可以点击AUTOSTART这样就可以让系统启动后自动开启frpc客户端了
